# Era 2009-2013 Replay

This POC runs curated historical boundary windows instead of replaying large chain ranges.

## Why curated windows

- Keeps data volume small and reproducible.
- Targets known transition boundaries where historical rule context matters.
- Preserves fast iteration for differential instrumentation.

Current anchors:

- BIP16 enforcement boundary: `173805`
- BIP34 buried activation boundary: `227931`

## Run

Offline fixtures only (default):

```powershell
cargo run -p jurassic-bitcoin-cli -- replay-era --manifest fixtures/manifests/era_2009_2013_poc.json --out-dir artifacts/era-2009-2013 --limit-per-epoch 200 --force
```

## Materialize RPC fixtures (optional pre-step)

Treat RPC fetch as fixture acquisition, not analysis.

Current workflow (until a dedicated `fetch-fixtures` command lands):

```powershell
cargo run -p jurassic-bitcoin-cli -- replay-era --manifest fixtures/manifests/era_2009_2013_poc.json --out-dir artifacts/era-2009-2013 --limit-per-epoch 200 --rpc-fetch --force
```

Then run replay again without `--rpc-fetch` for deterministic offline analysis:

```powershell
cargo run -p jurassic-bitcoin-cli -- replay-era --manifest fixtures/manifests/era_2009_2013_poc.json --out-dir artifacts/era-2009-2013 --limit-per-epoch 200 --force
```

Conceptually:

- materialization = acquire fixture samples
- replay = analyze fixture samples

## Fixture storage

- Small reproducible blobs live in git: `fixtures/blobs/*.json`.
- Manifest lives in git: `fixtures/manifests/era_2009_2013_poc.json`.
- RPC-fetched cache lives outside committed artifacts:
  - `JB_FIXTURE_CACHE` if set
  - else `fixtures/cache` (gitignored)

## Output layout

Replay writes one directory per era window:

- `artifacts/era-2009-2013/<window>/events/`
- `artifacts/era-2009-2013/<window>/replay-summary.json`
- `artifacts/era-2009-2013/<window>/summary.json`

Even when no divergences occur, both summary files are still produced.

## Interpreting summaries

- `replay-summary.json`: basic throughput (`checked`, `divergences`).
- `summary.json`: aggregated class/reason/mutation histograms.
- First quirk target in this POC uses synthetic fixture metadata (`checksighook`) to produce controlled script-side failures and nonzero `SCRIPT_FAIL` buckets.

## Oracle scope note

Current Core oracle uses `testmempoolaccept`, so epoch metadata is an organizational lens for stratified replay, not proof of full historical block-validation semantics at height `H`.
